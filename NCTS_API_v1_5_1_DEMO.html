Automated Vehicle Inspection Booking System

A sophisticated API solution that simplifies the car testing appointment process

Project Overview

The Challenge

In Ireland, car owners must schedule mandatory vehicle tests (NCT) through the national system. However:

  • Available slots are often booked weeks in advance
  • Desirable slots disappear quickly when they become available
  • Users need to constantly check the website for cancellations
  • The booking process is manual and time-consuming

My Solution

I developed a "concierge" system that automates the tedious parts of booking, while keeping users in control:

  • Automatically scans for available appointment slots
  • Temporarily holds a slot when found
  • Notifies the user immediately
  • Allows the user to complete the booking officially on the NCTS website

How It Works

This demo shows the full workflow of the system from searching for appointment slots to holding them for a customer:

✓ Smart Slot Search

Searches for available car test slots based on location and date preferences, using proxies to prevent rate limiting.

✓ Slot Validation

Quickly verifies if a specific slot is still available before attempting to reserve it.

✓ Automatic Slot Holding

Temporarily reserves a slot while the customer decides, making it visible only to them.

✓ Real-time Notifications

Instantly alerts the system/customer when a suitable slot is found and held.

✓ Resource Management

Efficiently releases held slots and cleans up resources when a reservation is completed or cancelled.

✓ Secure API Design

Implements API key authentication, webhook verification, and structured response formats.

Demo 1: Finding Available Appointment Slots

This demonstration shows how the system searches for available slots based on customer preferences:

Video shows the system searching for available slots across different test centers on specific dates.

The search process:

  1. Customer specifies their vehicle registration and preferred locations/dates
  2. System launches an automated browser session to search the official website
  3. All available slots are found and returned via webhook notification
  4. The client application receives a structured list of available slots

Key moments in the demo: 0:41 (Switching between centers), 1:00 (Processing dates), 1:20 (Receiving results)

Demo 2: Validating & Holding a Slot

Once a desirable slot is identified, the system validates its availability and holds it for the customer:

Video shows the system reserving and holding a slot for a specific vehicle registration.

The holding process:

  1. System validates the selected slot is still available
  2. Automated browser navigates the booking process using temporary information
  3. The slot is reached and held at the payment page
  4. A background process maintains the hold for ~30 minutes
  5. The customer is notified that a slot is being held for them

Key moments in the demo: 0:16 (Navigation start), 0:49 (Using temporary data), 1:15 (Reaching payment page), 3:13 (Proving the slot is held)

Key Business Benefits

  • Time Savings: Customers no longer need to constantly refresh the website looking for slots
  • Increased Success Rate: The automation can find and hold slots faster than manual searching
  • Customer Satisfaction: Users still complete their own booking, maintaining full control
  • Ethical Approach: The system works as a "concierge service" rather than bypassing the official platform
  • Scalability: Can handle multiple concurrent searches and holds efficiently
